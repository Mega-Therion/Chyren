Crystal clear diagnosis. This is a **one-line bug** — `RichLog.write()` in Textual does not accept an `end` keyword argument. The fix and the full corrected pattern are below.

## The Bug — Exact Line

```python
# LINE 229 — BROKEN
log.write("\n[bold #7c3aed]Ω ›[/bold #7c3aed] ", end=True)
```

`RichLog.write()` is not Python's `print()` — it has no `end` parameter. It writes one complete line per call. Passing `end=True` causes the `TypeError` that crashes your worker.

## The Fix — Two Patterns

**Option 1: Just drop `end=True`** (simplest, one character change):

```python
# Write the OmegA prefix as its own log entry
log.write("\n[bold #7c3aed]Ω ›[/bold #7c3aed]")
```

**Option 2: Buffer and write the full streamed response** (correct pattern for token streaming):

The right way to stream tokens into a `RichLog` is to accumulate the full response string and update the last written line. Here's the proper streaming pattern:

```python
async def stream_response(self, user: str) -> None:
    log = self.query_one("#chat-log", RichLog)
    inp = self.query_one("#user-input", Input)

    # Write user message first
    log.write(f"[bold #a78bfa]You ›[/bold #a78bfa] {user}")

    payload = {
        "user": user,
        "history": self.history,
        "voiceMode": False,
    }
    headers = {
        "Content-Type": "application/json",
        "x-omega-operator-token": self.token,
    }

    full_response = ""

    try:
        async with httpx.AsyncClient(timeout=120) as client:
            async with client.stream("POST", self.api_url, json=payload, headers=headers) as resp:
                # Write the OmegA prefix ONCE before streaming starts
                log.write("[bold #7c3aed]Ω ›[/bold #7c3aed] ", end="")  # ← still wrong if using RichLog
                
                # CORRECT: accumulate then write
                buffer = ""
                async for chunk in resp.aiter_text():
                    buffer += chunk
                    # Update in place by clearing and rewriting last line
                    # RichLog doesn't support in-place edit, so write chunks as they come:
                    self.call_from_thread(log.write, f"[bold #7c3aed]Ω ›[/bold #7c3aed] {buffer}")

    except Exception as e:
        log.write(f"[bold red]ERROR:[/bold red] {e}")
```

The cleanest approach for a streaming TUI is to use a `Static` or `Label` widget as the "live output" target and swap it to a `RichLog` entry only when the stream completes:

```python
# Streaming accumulator pattern — works perfectly with RichLog
full_response = ""
async for chunk in resp.aiter_text():
    full_response += chunk

# Write the complete response as one entry after streaming finishes
self.call_from_thread(
    log.write,
    f"[bold #7c3aed]Ω ›[/bold #7c3aed] {full_response}"
)
```
