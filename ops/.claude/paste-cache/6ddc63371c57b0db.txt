Here's a complete Bash script that will clone your **OmegA-Architecture** repository, install dependencies, patch the `web/next.config.ts` file to allow browser microphone access and widen the Content‑Security‑Policy (CSP), build the Next.js app, and deploy it to Vercel using the Vercel CLI.

I pulled the current `next.config.ts` from the repository and noticed that it explicitly disables microphone access and has a narrow `connect-src` whitelist. The script below replaces `microphone=()` with `microphone=(self)` and appends ElevenLabs and LiveKit host patterns to the `connect-src` directive. You can adapt those hosts to match whatever voice/TTS provider or LiveKit domain you actually use.

The web README notes that LiveKit variables are optional and can be enabled by setting the appropriate environment variables. I’ve included optional lines for them in `.env.local` so you can uncomment and fill them in when needed.

**Usage instructions**

1. **Set required environment variables** – Before running the script, export the following variables in your shell:

   * `OMEGA_API_BASE_URL` – base URL of your backend (e.g., `https://omega-sovereign.vercel.app`)
   * `OMEGA_API_KEY` – your API key if required (leave blank if not used)
   * `OMEGA_CHAT_PATH` – chat endpoint path (e.g., `/api/chat`)
   * `VERCEL_TOKEN` – your Vercel personal access token

   Optionally set voice‑related variables (`ELEVENLABS_API_KEY`, `NEXT_PUBLIC_LIVEKIT_URL`, etc.) if you plan to use ElevenLabs or LiveKit.  These variable names align with the repository’s documented environment variables.

2. **Make the script executable** – In a terminal, navigate to the directory where you saved the script and run:

   ```bash
   chmod +x omega_setup_deploy.sh
   ```

3. **Run the script** – Execute it from your terminal:

   ```bash
   ./omega_setup_deploy.sh
   ```

The script will:

* Check for and install any missing prerequisites (git, node, npm, curl, Vercel CLI).
* Clone or update `Mega‑Therion/OmegA‑Architecture`.
* Install Node dependencies inside the `web` folder.
* Patch `next.config.ts` to allow microphone access and extend the `connect-src` directive, replacing the default `camera=(), microphone=()` setting.
* Create a `.env.local` file with your core environment values and commented placeholders for voice/LiveKit settings.
* Build the Next.js app.
* Use `vercel pull`, `vercel build` and `vercel deploy` to push the prebuilt output to production.

After it completes, the deployment will be live on Vercel under the project tied to your token. Feel free to inspect or adjust the script before running, especially if you need to customize the CSP hosts for other providers.
