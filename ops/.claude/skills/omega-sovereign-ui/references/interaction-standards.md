# Interaction Standards

## Guiding Principles

- **Flow**: Transitions must be purposeful and direct.
- **Feedback**: Immediate visual confirmation of state changes (e.g., ADCCL gate approval).
- **HUD-Centric**: Information should be presented on the HUD in an orderly, prioritized fashion.
- **Minimalism**: If an item does not change state, do not highlight it.
- **Statement-Driven**: Use concise text for status updates (e.g., "AEON SYNCHRONIZED" rather than "The AEON layer has finished synchronizing the state...").
