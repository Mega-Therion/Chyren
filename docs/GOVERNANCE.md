## 📝 Contribution Process

### For External Contributors

1. **Read** [CONTRIBUTING.md](./CONTRIBUTING.md) fully before proceeding
2. **Open an Issue** to discuss the change before implementing
3. **Fork** the repository and create a feature branch
4. **Submit** a Pull Request using the PR template
5. **Pass** all CI checks (Rust, Python, Next.js)
6. **Review** by Sovereign Lead required for all merges

All contributions must:
- Pass ADCCL constitutional alignment checks
- Include appropriate tests
- Follow the existing code style and architecture
- Never compromise the Master Ledger integrity

### Merge Criteria

A pull request may be merged when:
- [ ] Constitutional alignment verified
- [ ] All CI workflows pass
- [ ] Sovereign Lead approves
- [ ] No regression in ADCCL threshold behavior
- [ ] Ledger integrity maintained

## 🔄 Versioning & Releases

Chyren follows [Semantic Versioning](https://semver.org/) (`MAJOR.MINOR.PATCH`):

| Version Type | When |
|-------------|------|
| `MAJOR` | Breaking changes to core API or ledger format |
| `MINOR` | New features, new provider spokes, new crates |
| `PATCH` | Bug fixes, security patches, documentation |

Releases are managed by the Sovereign Lead. The automated `release.yml` workflow generates changelogs from conventional commits.
